CAS PARTICULIER : navigateur Chrome

En raison d'une restriction de sécurité (Same Origin Policy), le navigateur Chrome empêche le chargement des requêtes Ajax si l'URI commence par file://.

Pour contourner cette restriction et travailler localement avec Chrome :

A) Avec un Mac

Lancez Chrome à partir d’un terminal :

  1. Ouvrez un terminal : Utilitaires/Terminal.
  2. Déplacez-vous dans le dossier de Grille. Par exemple, si JcruX est sur votre bureau :
    cd Desktop/jcrux
  3. Sélectionnez et copiez le code suivant (cmd + c) :
    open -a Google\ Chrome ./index.html --args --disable-web-security --allow-files-access-from-files
  4. Collez ce code dans le terminal (cmd + v) et validez-le avec la touche <Entrée>.
    Le navigateur Chrome s'ouvre à la page d'accueil.
  5. Cliquez sur le lien 'Générateur' du sommaire.

Etc.

Vous pouvez aussi rendre exécutable le script chrome_open_mac comme ceci :
chmod +x chrome_open_mac.sh

Et l’exécuter ainsi :
./chrome_open_mac.sh

B) Avec Windows

Ouvrez une invite de commandes CMD. La commande devrait ressembler à :
chrome.exe --disable-web-security --allow-files-access-from-files

Les retours d'expérience sont bienvenus. Je n’ai pas de machine Windows sous la main.

C) Autres méthodes

Principe : simuler un serveur HTTP local.

C-1) Environnement Unix/Mac OS X/GNU/Linux.

Si vous disposez d'une installation du langage Python sur votre ordinateur, vous allez procéder ainsi :

  1. Ouvrez un terminal et déplacez-vous dans le dossier Gentimeline.
  2. Exécutez le code :
    python -m SimpleHTTPServer
  3. Dans le navigateur Chrome, saisissez l'URL :
    http://localhost:8000/index.html

C-2) Environnement Windows

Vous devez installer un mini-serveur HTTP.
Cf. http://www.zmws.com/
